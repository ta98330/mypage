jQuery Mobile DOMキャッシュのせい？ (2016.1.19)

Mindfulnessアプリのログアウトボタンを押したとき，確認のポップアップを表示するようにした．しかしポップアップがうまく動作しないことがまれに起こった．調べたところ，原因として次のことが考えられる．Ajax通信によりページを現在の文書ツリーに取り込んだ上でおもむろに目的のページに移動するが，Ajax式リンクでは，新たなページに移動した後，不要になったページを適宜破棄する．

ログアウトボタンがあるアプリのホーム画面では，キャッシュ機能を有効にすることで，ページ遷移後もページを文書ツリーで維持できるDOMキャッシュを有効にしていた．しかし，アプリの挙動を見ると，改めてホーム画面に遷移した場合，新たにホーム画面の文書を現在の文書ツリーに取り込んでいた．そのためポップアップを定義したidが重複し，表示されなかった．

data-dom-cache="true"を消去し，DOMキャッシュを無効にしたところ，重複することはなくなり，ログアウトが正常にできるようになった．

PHP POST送信時 プロキシの罠 (2015.12.9)

このページを参考にPHPでフォームを介さずにPOSTデータを送信するプログラムを作成した．しかしローカル間で別のApacheのサーバーには送信できるが，hetemlサーバーのファイルに送信するときに「urlが存在しない」とエラーが出た．

原因が全くわからなかったが，自宅のPCでやってみたところ何のエラーも出ずhetemlに送信ができた．よって原因はプロキシであることがわかったので検索し，このページを参考にプログラムを書き換えると送信することができた．どうしてもうまくいかない場合は学校のプロキシ環境ではないところでテストしてみるといいかも．

CSS disabledで無効にしたボタンのデザイン (2015.11.30)

disabledで無効にしたボタンのデザインを変更するときに発生した問題．disabled属性をつけたボタンが自動で薄い灰色になって押せなくなるが，CSSでデザインを変更する．参考のとおりにUI要素状態擬似クラスで背景文字色は変えられたが全体的に薄いままだった．原因はopacityプロパティだった．この値が0.3になっているようだ．値を1に変更して無事解決した．

SQL MAX関数の条件 (2015.11.5)

最大値を求める場合に注意する（自分の認識不足だった）点．
条件の中で最大値を求めたかった．ここではidが1で，今日（$today）の時間がもっとも新しいデータが抽出したかった．まず自分が考えたSQLは，
SELECT * FROM mf_impressions WHERE id = 1 AND (datetime BETWEEN '$today 00:00:00' AND '$today 23:59:59') AND datetime = (SELECT MAX(datetime) FROM mf_impressions)
しかしこの場合もっとも新しいデータが得られるが，違う（1以外の）idのデータが最新の場合，該当データ無しとなる．最新のデータのidが1では無いためである．MAX関数は「グループ内の最大の値を返すもの」であり，WHERE id = 1で，idが1のデータがグループ化されたわけではない．そのためGROUP BY句を用いてグループ化する必要がある．

改良後のSQL
SELECT * FROM mf_impressions WHERE datetime = (SELECT max(datetime) FROM mf_impressions WHERE id = 1 AND (datetime BETWEEN '$today 00:00:00' AND '$today 23:59:59') GROUP BY id)
これで欲しい値を得ることができた．

jQuery mobile URLが表示されているページ内容と違う　(2015.10.5)

jQuery mobileで作ったページでページ内リンクが動作するときとしないときがあったので原因を探していたところ，発見した問題．

jQuery mobileのフォームのsubmit先で他のページにリダイレクトしたが，ブラウザのアドレスバーに表示されているURLがリダイレクト先のURLにならず，submit先のアドレスのままになってしまう問題が生じた．

data-role="page"としてるところにdata-urlも指定し，かつformタグにdata-ajax=“false”を指定することで解決する．jQuery Mobileがデフォルトで非同期submitするために起こる現象である．

HTTPSでCDNを利用する　(2015.9.30)

HTTPからHTTPSに変更した時にjQuery MobileをCDN経由で利用していた際，Chromeでこのような不具合が生じた．セキュリティが強化されhttpsのページに安全性を確認されていないコンテンツが表示できなくなったためである．CDNのURLをhttp:~からhttps:~に変更することで解決した．

MySQL datetime型カラムの部分利用 (2015.9.4)

MySQLのDATE_FORMAT関数を利用することで実現できる．
SELECT * FROM テーブル名 WHERE (DATE_FORMAT(カラム名, 'フォーマット文字') = '検索内容');
フォーマット文字について

例：『mf_impression』テーブルのdetatime型カラム『timing』のデータが09月04日のものを検索する．
SELECT * FROM mf_impressions WHERE (DATE_FORMAT(timing, '%m%d') = '0904');

XAMPPの設定メモ (2015.6.12)

XAMPPインストール後の設定　追記：（2015.8.20）v1.8.2-6では不要.

php.ini

XAMPPのコントロールパネルを開いて「Apache」の Config ボタン → PHP → 開いたフォルダの中の php.ini

以下の8つを探して、それぞれ文頭の ; を外す.
;mbstring.language = Japanese
;mbstring.internal_encoding = EUC-JP
;mbstring.http_input = auto
;mbstring.http_output = SJIS
;mbstring.encoding_translation = Off
;mbstring.detect_order = auto
;mbstring.substitute_character = none
;mbstring.func_overload = 0

mbstring.internal_encoding = EUC-JP を mbstring.internal_encoding = UTF-8 に変更.
mbstring.http_output = SJIS を mbstring.http_output = UTF-8 に変更.
mbstring.encoding_translation = Off を mbstring.encoding_translation = On に変更.

SQLでサブクエリ (2015.5.28)

SELECT * FROM member WHERE id = (SELECT id FROM user_logs WHERE date = '2015-05-27' AND end_time IS NULL);
括弧の前がイコールでは，値が複数の場合処理できない．イコールをINにすれば解決．
SELECT * FROM member WHERE id IN (SELECT id FROM user_logs WHERE date = '2015-05-27' AND end_time IS NULL);

Google カレンダーをホームページに埋め込む際予定が反映されなかった (2015.5.20)

Chromeで作業する際，埋め込みたいカレンダーがあるgoogleアカウントとは別のアカウントでChrome自体にログインしている場合に起こった問題．

このページの説明の通りGoogleカレンダー埋め込み支援ツールまで進むと，Chromeにログインしている方のアカウントに自動で切り替わってしまう．そのため，そこで生成されたiframeコードは埋め込みたいカレンダーのものとは別のコードが表示されていてうまくいかない．今回は別のブラウザで作業することで回避．

phpで作った卒検支援ページをローカルからhetemlサーバーへあげるときに起きた問題 (2015.5.19)

ローカル環境では動いていたが論理演算子でエラーが発生したためコードを変更．

                        if(!empty($_POST['come'] or $_POST['out'])) //ローカルでは動いたがhetemlはエラー
                        


                        if(!empty($_POST['come']) or (!empty($_POST['out']))) //これならばhetemlでも動いた

また，下で記述したエラー制御演算子＠「アットマーク」もhetemlではエラーになった．

(追記：2015.05.21)ミーティングで，そもそもemptyの中で論理演算子というのはいかがなものかということで調査．
empty()は引数に指定した変数に値がセットされているかどうか調べる言語構造の一部で，0か空ならTRUE，それ以外はFALSEを返すというもの．この説明からするとやはりemptyの引数の中で論理演算をするというのは間違っているように思える．

PHPでの＠「アットマーク」の意味 (2015.5.11)

PHPの＠「アットマーク」は「エラー制御演算子」といい，エラーがあってもエラーメッセージを出力させないようにするためのもの．

SQLでNULLのレコードを削除したい (2015.5.11)

delete from テーブル名 where レコードのフィールド（idなどを指定）=削除するレコード値で削除するレコード値をNULLで実行したところうまくいかなかった．調べたところNULLとの比較はできないとのこと．この場合はレコードのフィールド（idなどを指定） IS NULLまたはIS NOT NULLで行う．
